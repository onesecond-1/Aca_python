불 자료형
-값이 true, false
-true, false를 자료형으로도 나타낼 수 있는데 true는  자료형에 값이 존재하는 경우, false는 자료형에 값이 존재하지 않는 경우
ex)false: 0, "", []

집합 자료형 함수
-add(값): 집합에 값을 추가한다. (단, 중복된 값은 추가되지 않는다.)
-remove(값): 해당 값을 집합에서 삭제한다. (단, 없는 값일 경우 오류 발생)
-pop(값): 해당 값을 삭제하고 반환한다.

연산자 우선 순위
-최우선 연산자: 소괄호() 점.
-단항 연산자: 더하기+ 빼기-
-산술 연산자: 사칙연산 + - * / // % **
-쉬프트 연산자: <<(왼쪽 쉬프트), >>(오른쪽 쉬프트)
ex) 10 = 1010(2진수), 10<<1 = 10100(2진수)
    즉, 자릿수를 밀고 0을 추가.
    계산하면 왼쪽 쉬프트는 *2, 오른쪽 쉬프트는 /2
-관계 연산자: 부등호== !=(다름) > < >= <=, 의문문으로 출력 시 불 자료형으로 표현됨.
-논리 연산자: 여러 조건을 비교하여 하나의 불 자료형으로 나타내는 연산자
and: and를 기준으로 양 옆에 조건들이 오는데 두 조건들이 참이어야 결과는 참.
or: or를 기준으로 양 옆에 조건들이 오는데 두 조건중 하나만 참이어도 결과는 참.
not: 불 자료형의 결과를 반전.
-삼항 연산자: `(조건식) ? 참 : 거짓`
            물음표 왼쪽에 있는 조건식의 결과가 참이라면 콜론 왼쪽에 있는 값을 반환하고, 조건식의 결과가 거짓이라면 콜론 오른쪽에 있는 값을 반환한다.
            `조건식의 결과가 참일 경우` if 조건식 else `조건식의 결과가 거짓일 경우`의 형태로 표현.
-대입연산자: 등호, 변수에 값을 저장

딕셔너리 자료형
-값을 중심으로 양 옆에 중괄호가 온다.
-딕셔너리 자료형의 값은 일반적인 자료형과 다르게 키(key)와 값(value)이 한 쌍을 이루어서 값을 표현한다.(딕셔너리 고유 특징)
-딕셔너리의 key는 인덱스를 대체한다. 그러나 순차성은 존재하지 않는다.
-key는 value를 대표하는 값이다.
-key는 기본 자료형으로 표현하고, value는 모든 자료형으로 표현 가능.

딕셔너리 자료형 함수
-get(키): 해당 키의 값을 반환한다.
-keys(): 딕셔너리 안에 저장되어있는 key들을 전부 반환한다.
-values(): 딕셔너리 안에 저장되어있는 value들을 전부 반환한다.
-items(): 딕셔너리 안에 저장되었는 key와 value를 하나의 튜플로 묶어서 반환한다.

제어문
-어떤 조건에 의해 프로그램의 흐름을 바꾸는 기능

제어문의 종류
-조건문: if문, 상항연산자가 존재한다. / 특정 조건의 만족 여부에 따라 실행한다.
-반복문: while문, for문이 존재. / 특정 코드를 반복시키는 기능.
-기타 제어문: break을 활용하여 제어문을 중단시키는 기능. / continue는 코드를 건너뛰고, 다시 반복. / pass는 아무런 작업도 하지 않을 경우 추가.

조건문
-조건식의 결과(불자료형)에 따라 실행되는 구문이 달라진다.
-if, if~elif, if~elif~else, if~else모양으로 만들 수 있다.
-조건문은 if문이 대표. 삼항연산자는 비중이 적다.(삼항연산자는 조건이 하나만 들어갈 수 있기 때문)

조건문의 생김새

    수행문 *꼭 들여쓰기해야 실행가능.
