클래스
-여러 변수와 함수(메서드)를 묶어서 처리할 수 있는 것.
-공통된 요소들이 모여있는 하나의 틀, 설계도

클래스 vs 객체
클래스: 붕어빵 틀. 객체를 만드는 기틀
객체: 붕어빵. 즉, 클래스로 인해 만들어지는 결과

클래스 선언
class (클래스 이름):
    # 메서드1 정의
    def 메서드1 이름(self, 매개변수, etc.):
        self.변수이름=값    # 클래스 멤버 변수(클래스에 소속된 변수)
        수행문
    # self는 클래스를 의미함.
    # 메서드2 정의
    def 메서드2 이름(self, 매개변수, etc.):
        수행문
        .
        .
        .
-self: 객체 구별용 매개 변수, 클래스에 속해있는지 알 수 있다.
-클래스 이름은 대문자로 시작한다.
-데이터 속성: 멤버 메서드, 멤버 변수(ex.self.변수이름의 형태로 만든 변수) 등 클래스의 수행문에서 만들어지는 요소들을 의미.

클래스의 활용
-클래스는 추상적인 개념이므로 객체를 만들어서 사용해야 한다. 이것을 객체화라고 한다.
-객체화 방법: 객체 이름=클래스 이름()
-객체화 후 각 멤버에 접근하는 방법
: 객체 이름.변수 이름
: 객체 이름.메서드 이름()

클래스를 사용하는 이유
- 하나의 클래스를 활용하여 여러 개의 객체를 만들고 그때마다 다르게 처리할 수 있게끔하여 코드의 반복을 줄일 수 있게 때문에 클래스를 사용.

생성자(Constructor)
-객체화 시 실행되는 메서드.
-클래스를 활용하여 객체를 생성하는 순간 호출되는 메서드.
-생성자는 주로 클래스의 멤버 변수를 선언하고 초기화 시에 사용.
-생성자 선언 시 메서드 이름을 __init__라고 명칭.
-클래스 내에서 임의로 선언한 생성자가 존재하지 않다면, 파라미터를 아무것도 받지않는 기본 형태의 생성자가 생략되어 정의되어 있다.
